# 需求文档

## 1. 应用概述

### 1.1 应用名称
HSK智能学习平台

### 1.2 应用描述
面向外国人的HSK学习平台,通过AI驱动的批改和分析功能,实现「诊断→分析→治疗」的教学闭环。平台支持中文/俄语/英语三语切换,切换时双语显示,提供作文批改、作业批改、口语分析、成绩分析、学习技巧和历史记录六大功能模块。

## 2. 用户与使用场景

### 2.1 目标用户
- 学习HSK的外国学生
- HSK教师(用于批改学生作业和分析学习情况)

### 2.2 核心使用场景
- 学生提交作文/作业/口语录音,获取AI批改结果和针对性练习
- 学生上传成绩数据,查看学习趋势和薄弱环节
- 学生根据历史弱项获取个性化学习计划
- 教师批量处理学生作业并追踪学习进度

### 3.2 登录/注册页面

#### 3.2.1 注册
- 用户输入用户名、密码、确认密码完成注册
- 选择母语(俄语/英语)
- 使用Supabase Auth实现,用户名注册时内部用username@hsk-yu.app作为合成邮箱存储
- 注册成功后自动登录

#### 3.2.2 登录
- 支持两种登录方式:
  - 用户名+密码登录
  - 邮箱+密码登录(已有邮箱账号)
- 使用Supabase Auth实现
- 登录成功后自动检测该账号是否已绑定小Yu码
- 若未绑定小Yu码,弹出小Yu码绑定弹窗
- 若已绑定小Yu码,进入首页

### 3.3 小Yu码绑定弹窗

#### 3.3.1 触发条件
- 用户首次登录/注册后,账号未绑定小Yu码时自动弹出

#### 3.3.2 功能说明
- 保留现有UI设计,优化弹出动画
- 用户输入小Yu码
- 系统验证小Yu码有效性:
  - 有效:将码与账号永久绑定(invitation_codes.bound_user_id = auth.uid()),进入首页
  - 无效:提示用户重新输入
- 一个码只能绑定一个账号,已绑定的码不可被其他人使用

## 4. 业务规则与逻辑

### 4.1 账号系统与认证
- 使用Supabase Auth实现用户认证
- 用户名注册时,内部用username@hsk-yu.app作为合成邮箱存储
- 支持用户名+密码和邮箱+密码两种登录方式
- 登录成功后,Supabase自动维护session,下次访问自动登录

### 4.2 小Yu码绑定逻辑
- 小Yu码为唯一激活/邀请码
- 用户首次登录/注册后,系统检测账号是否已绑定小Yu码
- 若未绑定,弹出小Yu码输入界面
- 用户输入有效码后,系统将码与账号永久绑定(invitation_codes.bound_user_id = auth.uid())
- 一个码只能绑定一个账号,已绑定的码不可被其他人使用
- 下次访问自动登录,无需再次输入码

### 4.3 路由保护逻辑
- 未登录用户访问功能页面:自动跳转至登录页面
- 已登录但未绑定小Yu码:弹出小Yu码绑定弹窗
- 已登录且已绑定小Yu码:正常访问所有学习功能
- 管理员后台(/yuteacher)需密码验证后才能访问

### 4.4 AI批改与分析
- 通过Edge Function调用AI实现批改和分析功能
- 作文批改和作业批改:AI分析词汇、语法、结构、表达、逻辑、书写六个维度
- 口语分析:AI分析发音、声调、流利度三个维度
- 成绩分析:AI根据成绩数据生成趋势图、雷达图和学习建议

### 4.5 雷达图交互逻辑
- 用户点击雷达图某个维度时,下方错误详情区域自动筛选并高亮显示该维度的所有错误
- 未点击时,默认显示所有维度的错误

### 4.6 针对性练习生成逻辑
- 系统根据雷达图中得分最低的1-2个维度生成针对性练习
- 练习类型包括改错题和造句题

### 4.7 历史记录沉淀逻辑
- 作文批改、作业批改、口语分析、成绩分析的所有结果自动保存至历史记录
- 每条记录包含:提交时间、模块类型、输入内容、批改结果、雷达图数据、练习题

### 4.8 学习技巧生成逻辑
- 系统自动提取用户历史记录中所有雷达图的薄弱项(得分低于60分的维度)
- 统计各维度出现频率,生成个性化学习计划和针对性练习题

### 4.9 多语言切换逻辑
- 用户选择语言后,界面文字双语显示:选中语言在前,中文在后
- 示例:选择俄语时,按钮显示为「Отправить 提交」
- 用户语言偏好保存至用户账户,下次登录自动应用

### 4.10 后端数据存储
- 使用Supabase作为后端,实现数据库存储、文件存储和用户认证
- 用户数据、历史记录、批改结果均存储在Supabase数据库中
- 上传的图片、录音文件存储在Supabase存储中
- 小Yu码信息存储在invitation_codes表中,包含码、绑定用户ID、今日调用次数、总调用次数、到期日期、状态等字段

### 4.11 我的学习舱数据计算逻辑
- 学习天数:从用户首次提交批改任务的日期开始计算至当前日期
- 总批改次数:统计用户所有批改任务(作文批改、作业批改、口语分析)的累计次数
- 综合平均分:计算用户所有批改结果的平均得分
- 能力维度雷达图:综合用户所有批改结果中各维度的平均得分
- 成长曲线:按时间顺序展示用户综合得分的变化趋势
- 高频错误归因:统计用户历史记录中各错误类型的出现频率,按频率降序排列
- AI个性化建议:基于用户历史数据和薄弱项生成学习建议

### 4.12 管理员后台权限管理逻辑
- 管理员可生成新小Yu码,系统自动生成唯一码
- 管理员可查看所有小Yu码的使用情况和绑定账号信息
- 管理员可修改用户的到期时间、每日调用上限、启用/禁用账号
- 用户账号被禁用后,无法访问学习功能
- 用户到期后,无法继续使用批改和分析功能
- 用户每日调用次数达到上限后,当日无法继续使用批改和分析功能

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户未登录访问功能页面 | 自动跳转至登录页面 |
| 用户已登录但未绑定小Yu码 | 弹出小Yu码绑定弹窗 |
| 用户输入无效小Yu码 | 提示用户小Yu码无效,请重新输入 |
| 用户输入已被绑定的小Yu码 | 提示用户该小Yu码已被使用,请联系管理员 |
| 用户账号被禁用 | 提示用户账号已被禁用,请联系管理员 |
| 用户账号已到期 | 提示用户账号已到期,请联系管理员续费 |
| 用户每日调用次数达到上限 | 提示用户今日调用次数已达上限,请明天再试 |
| 图片上传失败 | 提示用户重新上传,支持重试 |
| 录音上传失败 | 提示用户重新上传,支持重试 |
| AI批改服务异常 | 提示用户稍后重试,不保存本次提交 |
| 历史记录为空 | 提示用户暂无历史记录 |
| 学习技巧页面无薄弱项数据 | 提示用户先完成批改或分析任务 |
| Excel/CSV文件格式错误 | 提示用户检查文件格式并重新上传 |
| 雷达图数据缺失 | 显示默认雷达图,提示数据不完整 |
| 我的学习舱无历史数据 | 显示默认值(学习天数0天、总批改次数0次、综合平均分0分),提示用户开始学习 |
| 最近学习记录不足5条 | 显示实际记录数量 |
| 管理员后台密码错误 | 提示密码错误,请重新输入 |

## 6. 验收标准

1. 用户使用用户名和密码注册新账号,注册成功后自动登录
2. 系统检测到用户未绑定小Yu码,弹出小Yu码绑定弹窗
3. 用户输入有效小Yu码,系统将码与账号绑定,进入首页
4. 用户选择「作文批改」功能,粘贴一段中文作文并提交
5. 系统展示雷达图,标注薄弱项为红色
6. 用户点击雷达图中的「语法」维度,下方自动筛选显示语法错误详情
7. 用户查看左右对照的原文批改建议
8. 用户完成系统生成的针对性练习题
9. 用户进入「我的学习舱」页面,查看学习天数、总批改次数、综合平均分等核心指标
10. 管理员访问/yuteacher路由,输入密码进入管理员后台
11. 管理员生成新小Yu码,查看所有小Yu码的使用情况
12. 管理员修改某个用户的到期时间和每日调用上限
